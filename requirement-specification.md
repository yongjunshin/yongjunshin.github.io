# Requirement Specification
이 요구사항 문서는 CV 홈페이지 refactoring + migration + improvement를 위한 요구사항 초안이야.
이 요구사항은 상위 사용자가 경험하는 요구사항만 담고, 그 내부의 기술적인 구현 요구사항은 담고있지 않으니, 기술적 요구사항은 따로 분석해서 개발해야함.

## 메인 페이지

- 이름 출력
- 사진 출력
- 개인 소개 글 출력
- News 섹션에 최근 5개 출력 (날짜와 글이 출력됨)
- Highlihged publications 섹션에 highlight로 내가 마크해 놓은 논문들이 출력됨 (출력되는 형태는 publications 페이지에서 출력되는 방식과 동일)
- 외부의 개인 링크 연결 버튼들 (이메일, orcid, google scholar, github, linkedin)
- 모든 페이지 하단에는 copyright

## CV 페이지

- 내가 따로 관리해서 업데이트하고있는 CV PDF 버전을 열어주는 버튼
- General Information 섹션 출력 (기존 컨텐츠 모두 포함)
- Employment 섹션 출력 (기존 컨텐츠 모두 포함)
- Education 섹션 출력 (기존 컨텐츠 모두 포함)
- Honors and Awards 섹션 출력 (기존 컨텐츠 모두 포함)
- Academif Service 섹션 출력 (기존 컨텐츠 모두 포함)
- Publication, Patent, Projects 는 다른 탭으로 보내기
- 모든 섹션에서 기간 표시를 일관된 위치의 스타일이 일관되도록
- 특정 섹션으로 바로 이동하는 버튼들은 불필요하니 삭제해서 단정되고 간결하도록.

## Publications 페이지

- 연도별 그룹핑은 하지말고 시간 역순 정렬만 잘 되어 출력되면 됨.
- 각 논문의 정보 출력
  - 제목, 저자 (저자에서 내 이름은 밑줄.), 출판지 정보, 연도 출력.
  - 논문의 preview 이미지 출력
  - bib 복사 가능 버튼
  - pdf 여는 버튼
  - website로 보내는 버튼
  - slide 여는 버튼
- 논문 컨텐츠 추가할 때는 bib 형태로 이전과 동일하게 추가할 수 있도록

## Patent 페이지

- 이전과 동일한 특허 정보들이 깔끔하게 출력되도록

## Proejcts 페이지

- 현재/과거 별 그룹 없애고 그냥 시간 역순으로 정렬이면 충분.
- 카드형 디자인 말고, 그냥 리스트형태로 하고, 프로젝트 이름을 누르면 프로젝트 소개 페이지로 이동되도록
- 프로젝트 소개페이지는 기존에 적혀있는대로 마크다운으로 쉽게 작성하고, 그 컨텐츠가 그대로 보일 수 있도록 지원.

## Rositories

- 이 페이지는 삭제. 필요 없음.

## 그 외 요구사항

- light/dark 모드는 필요 없고, 하나의 깔끔한 디자인
- 디자인의 톤은 light mode이고, 모던한 computer scientist의 CV 홈페이지 스타일로.
- 전체 페이지가 일관된 스타일링을 유지하고 전문 디자이너가 디자인 한 것 처럼 높은 수준의 퀄리티로.
- 내부 코드를 나는 자세히 모르겠지만, 홈페이지 유지보수를 위해 아래의 명확한 폴더 구분 제공. (이게 기존 프로젝트에서 최악이었음. 컨텐츠바꿀때마다 바꿔야하는 파일 위치들 다 제각각이라 매번 기억안나서 힘들었음.)
  - contents: 각 페이지에 출력되는 컨텐츠 (markdown, bib, pdf, jpg, 등)만 넣는 위치. 대부분의 홈페이지 관리에서는 이 폴더만 접근할 예정. 컨텐츠가 많을 수 있으니 우리 홈페이지의 각 페이지를 sub-directory로 나눠줘.
  - configs: 아래의 코드 및 우리 홈페이지의 일부 내가 바꿀 수 있는 설정 값들은 모두 여기에서 쉽게 설정. 코드를 모르는 일반 사용자가 쉽게 다뤄서 제어해도되는 홈페이지의 특정 요소나 설정 규칙들만 선정해서 쉽게 설정할수있는 수준으로만 여기에 오픈해줘. 설정의 정말 대부분은 너가 알아서 판단하되, 일반사용자가 본인의 홈페이지를 유지하면서 CV 컨텐츠 추가와 더불어 유용하고 쉽게 다룰만한 설정들 정도만 여기에서 제어. (예를들어 컨텐츠의 간격 같은건 여기 오픈되면 안되고 너가 알아서 잘 만들어서 코드에 넣어야하는데, 밑줄을 쳐야하는 본인의 이름 텍스트를 변경할수있게하던가, 이런 일반사용자 레벨에서 쉬운 설정들만 포함)
  - code (여긴 나는 절대 안열어볼거고 폴더이름 달라도 됨.)
